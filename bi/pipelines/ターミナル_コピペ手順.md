# ターミナルにコピペするコマンド（Windows / PowerShell）

**使う場所**: Cursor や VS Code の「ターミナル」、または Windows の PowerShell。  
**前提**: 下のフォルダがあなたのPCのプロジェクトの場所です。違う場所に置いている場合は、`cd` の行だけ書き換えてください。

```
c:\Users\mizuk\python_investment
```

---

## ① 最初に1回だけ（ライブラリのインストール）

**必ず先に `cd` の行から実行してください。**  
`C:\Users\mizuk` のような**別フォルダにいるまま** `pip install -r requirements-movers.txt` だけ実行すると、  
`Could not open requirements file` になります（ファイルは `python_investment` の中にしかありません）。

ターミナルを開いたら、**次の2行をそのまま順にコピペ**して Enter。

```powershell
cd c:\Users\mizuk\python_investment
python -m pip install -r requirements-movers.txt
```

**どうしても `cd` を使わない場合**（フルパス指定）:

```powershell
python -m pip install -r c:\Users\mizuk\python_investment\requirements-movers.txt
```

（`python` で動かない場合は `py -m pip install -r ...` を試してください。）

---

## ② API キー（最初に1回＋キーを変えたとき）

`.env` というファイルにキーを書きます。

1. エクスプローラーで `c:\Users\mizuk\python_investment` を開く  
2. `.env.example` をコピーして、名前を **`.env`** に変える（メモ帳で編集してもよい）  
3. `.env` を開き、`JQUANTS_API_KEY` / `OPENAI_API_KEY` / `TAVILY_API_KEY` の右側に、取得したキーを貼る  

ターミナルからコピペでコピーだけする例（すでに `.env` がある場合は**上書きしない**でください）:

```powershell
cd c:\Users\mizuk\python_investment
copy .env.example .env
notepad .env
```

メモ帳が開いたらキーを保存して閉じます。

---

## ③ 値動きの大きい銘柄リストを作る（J-Quants）

**毎回、まずプロジェクトフォルダに移動**してから実行します。

```powershell
cd c:\Users\mizuk\python_investment
python scan_daily_movers.py
```

終わると `data\processed\` に  
`movers_YYYY-MM-DD.csv` と `movers_YYYY-MM-DD.json` ができます。  
**日付は実行した日のデータに依存**するので、ファイル名の日付をメモしてください。

---

## ④ 材料レポート（Markdown）を作る

**`③` でできた JSON のファイル名**に合わせて、次の `--input` の部分だけ書き換えてコピペします。

例（ファイル名は実際の名前に変えてください）:

```powershell
cd c:\Users\mizuk\python_investment
python report_mover_reasons.py --input data\processed\movers_2025-03-21.json --max-stocks 5
```

- `--max-stocks 5` … 先頭から最大5銘柄だけ処理（お試し・節約用。外せば制限なしに近い動きになります）

銘柄を直接指定する例:

```powershell
cd c:\Users\mizuk\python_investment
python report_mover_reasons.py --code 7203 --company-name トヨタ自動車 --as-of 2025-03-21
```

レポートは `reports\` に `.md` で保存されます。

## よくあるつまずき

| 状況 | 対処 |
|------|------|
| `Could not open requirements file` | **いまいるフォルダが違います。** 先に `cd c:\Users\mizuk\python_investment` を実行するか、上のフルパス版 `pip` を使う |
| `No matching distribution found for jquantsapi` / `jquants-api-client` | **パッケージ名の誤りを修正済み**です。`requirements-movers.txt` を保存し直してから `pip install -r ...` をやり直してください。それでも `versions: none` のときは **Python が古い**可能性があります。ターミナルで `python --version` とし、**3.10 以上**を推奨（3.9 以下では古いクライアント版のみ入ります） |
| `JQUANTS_API_KEY が未設定` | `.env` に J-Quants のキーがあるか確認 |
| `TAVILY_API_KEY` / `OPENAI_API_KEY` のエラー | `.env` に両方入っているか確認 |
| `python` が認識されない | `py` に置き換えて試す（例: `py scan_daily_movers.py`） |
| `--input` のファイルがない | `③` を先に実行し、`data\processed\` 内の実際の `movers_....json` 名を使う |
| `make_screening_master_v2` の財務取得 | **`/fins/summary` は `code` のみ**（4桁→5桁末尾0）。**日付さかのぼりはしない**（速さ優先）。`code` で 0 件の銘柄は欠損のまま。**スタンダードプランでも過去数年分は取得可能**だが、**銘柄によって API 上の開示履歴の長さが違う**（例: **130A** は `code=130A0` で **2024-02-13 開示が最古**で件数も少ない一方、**7203** は **2016 年**まで遡る）。株探の長年表と一致しないのは「プラン不足」ではなく **当該銘柄のサマリに古い年が載っていない**ことが多い。確認は `python inspect_fins_summary.py 銘柄` |
| `make_screening_master_v2` を回したのに Excel が更新されない | **既定で `.xlsx` も出力**（`--no-excel` を付けたときだけ省略）。ファイルが Excel で開きっぱなしだと書けずスキップされるので閉じて再実行。parquet だけ欲しいときは `--no-excel` |
| グロース（例: **130A**）や一部銘柄で財務が欠損・Kabutan とズレる | 実績は **4Q / FY 行の数値だけ**（中間四半期は使わない）。通期の OP/NP・売上は **同一開示に揃え、CurPerType は FY を 4Q より優先**（短信・株探の通期に近い。FY 行が無いときだけ 4Q）。**売上は FY/4Q の Sales のみ**（四半期累計の誤採用防止）。**IFRS は経常利益が空**のことがある。欠損は `*_data_gaps.parquet` で確認 |
| J-Quants の `/fins/summary` が **件数少なめ**（履歴が薄い） | 既定で **`yfinance_statement_fallback.py`** により **Yahoo の年次 `income_stmt` 等で売上・OP・利益・自己資本比率・株数を補完**（直近2期）。**翌期予想**は J-Quants 集約のまま。無効: `YFINANCE_STATEMENT_FALLBACK=0` または `--no-yfinance-statements`。要 `pip install yfinance` |
| Excel で **科学記数法** や日付が **`########`** | `convert_to_excel` で **千桁区切り・列幅**を付与。古い `.xlsx` ならマスタ再生成後に開き直す |

## 毎日のおすすめの流れ（短く）

```powershell
cd c:\Users\mizuk\python_investment
python scan_daily_movers.py
python report_mover_reasons.py --input data\processed\movers_実際の日付.json --max-stocks 10
```

2行目の `movers_実際の日付.json` は、①の実行後にフォルダにできたファイル名に合わせて書き換えてください。
