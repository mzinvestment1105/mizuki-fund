# Mizuki Fund コマンドリファレンス

**対象**: ターミナル（PowerShell）にコピペして使う用。  
**前提フォルダ**: 各コマンドの先頭に `cd` 行がある場合は、**必ずそこから実行すること**。

---

## 0. 最初に1回だけ（ライブラリのインストール）

```powershell
cd "C:\Users\mizuk\2026年 investment\Mizuki Fund\bi\pipelines"
python -m pip install -r requirements.txt
```

---

## 1. 毎朝の作業 ── GitHub から最新ETL結果を取得してExcelに反映

GitHub Actionsが毎日19:00に自動でスクリーニングマスターを更新している。  
それをローカルに落としてExcelにする。

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\mizuk\2026年 investment\Mizuki Fund\bi\pipelines\ops\run_daily_pull_and_excel.ps1"
```

**何が起きるか**:
1. GitHubから最新の `screening_master.parquet` 等をpull
2. parquet → `.xlsx` に変換
3. `bi/outputs/screening_master.xlsx` が更新される

> **Excelを開きっぱなしにしていると失敗する。先に閉じること。**

---

## 2. スクリーニングマスターをローカルで手動更新

GitHub Actionsが動かなかった・今すぐ最新にしたいときに使う。  
J-QuantsのAPIキーが必要（`.env` に設定済みであること）。

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\mizuk\2026年 investment\Mizuki Fund\bi\pipelines\ops\run_make_screening_master_v2.ps1"
```

**何が起きるか**:
1. J-Quants APIから全銘柄の株価・財務・信用残等を取得
2. `bi/outputs/screening_master.parquet` を更新（30〜60分かかる）
3. Excel変換は含まない → 終わったら手順1を実行

## 3. Deep Diveリサーチ（銘柄調査）

### ステップ1: EDINETから有報を取得する

```powershell
cd "C:\Users\mizuk\2026年 investment\Mizuki Fund\bi\pipelines"
python deep_dive.py --code 7256
```

- `7256` の部分を調べたい銘柄コードに変える
- 出力先: `research/stocks/7256_YYYY-MM-DD_data.md`

四半期報告書が欲しい場合:

```powershell
python deep_dive.py --code 7256 --type quarterly
```

---

### ステップ2: Perplexityでリサーチする（手動）

Perplexityでリサーチした結果をMarkdownファイルとして保存する。

**保存先・ファイル名のルール:**

```
research/stocks/7256_YYYY-MM-DD_perplexity_stock.md   ← 個別銘柄リサーチ
research/stocks/7256_YYYY-MM-DD_perplexity_sector.md  ← セクターリサーチ
```

（日付は今日の日付。例: `7256_2026-04-05_perplexity_stock.md`）

---

### ステップ3: Claude Code にファイルを渡してレポートを依頼する

コマンド不要。この会話（Claude Code）に対して：

1. 「Deep Diveレポートを作って」と言う
2. Claude が自動でファイルを読んでレポートを生成する

## よくあるトラブル

| 症状 | 対処 |
|------|------|
| `cd` したつもりなのにエラー | フォルダパスにスペースが入っているので **ダブルクォートで囲む** |
| Excelが更新されない | **Excelを閉じてから** もう一度実行する |
| `JQUANTS_API_KEY が未設定` | `bi/pipelines/.env` にキーを記入する |
| `python` が認識されない | `py` に置き換えて試す |
| ETL手動実行が30分以上かかる | 正常。全3700銘柄を処理しているため時間がかかる |
