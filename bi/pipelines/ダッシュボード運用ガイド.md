## 1. サイドバー3項目の機能

## `dashboard app`（ホーム）

- 役割: ダッシュボードの入口ページ
- 主な表示:
  - このダッシュボードの目的
  - データ本体（`data/processed/screening_master.parquet`）を使うこと
  - 自動更新が平日19:00想定であること
- 実ファイル: `dashboard_app.py`

## `値幅レポート_画像`

- 役割: スクリーンショット画像から銘柄を抽出し、材料レポート（Markdown）を生成
- 画面でできること:
  - 画像アップロード（PNG/JPG/WebP）
  - 基準日の指定
  - モデル指定（Vision/Chat）
  - レポート生成とダウンロード
- 内部処理:
  1. 画像を OpenAI へ送って銘柄抽出
  2. Tavily で関連ニュース検索
  3. OpenAI で日本語レポート生成
- 実ファイル: `pages/2_値幅レポート_画像.py`

## `screening master`

- 役割: 毎日生成されるスクリーニング統合データを表形式で閲覧
- 画面でできること:
  - 行数・最終更新日時の確認
  - 「最新データを読み直す」ボタンで再読込
  - 表示上限行数の調整
  - `MarketCap` 降順表示（列がある場合）
- 実ファイル: `pages/3_screening_master.py`

## 2. このツールが何で作られているか

## フロント（画面）

- Streamlit
- 構成:
  - `dashboard_app.py`（ホーム）
  - `pages/*.py`（サブページ）

## データ生成（バッチ）

- Pythonスクリプト: `make_screening_master_v2.py`
- 主なデータソース: J-Quants API v2（`JQUANTS_API_KEY`）
- 出力:
  - `bi/outputs/screening_master.parquet`（本体）
  - `bi/outputs/*_data_gaps.parquet`（財務欠損確認）
  - `bi/outputs/*.xlsx`（閲覧用。parquet から変換）

## 自動実行（推奨構成）

- **GitHub Actions**: 平日19:00 JSTにETLを実行し、`bi/outputs/` を更新
- **Windows Task Scheduler（PC側）**: 平日19:10 JSTに `git pull` で取り込み、parquet → xlsx 変換まで実施
  - 実行スクリプト: `bi/pipelines/ops/run_daily_pull_and_excel.ps1`

## 同時読み取り対策

- `make_screening_master_v2.py` は parquet を
  - 一時ファイルに書く
  - 最後に rename（置換）
  の順で保存
- これにより、ダッシュボードが更新途中ファイルを読むリスクを下げる

## 3. 更新方法を踏まえた日次の使い方

## 基本フロー（平日）

1. 19:00に GitHub Actions がETLを実行（出力更新）
2. 19:10に PC側が `git pull` で取り込み、Excel変換
3. ダッシュボードを開く
3. `screening master` ページで「最新データを読み直す」
4. 行数・更新日時を確認して閲覧

## 手動で即時更新したいとき

```powershell
cd "C:\Users\mizuk\2026年 investment\Mizuki Fund"
git pull --rebase
python "bi\pipelines\convert_to_excel.py" --input "bi\outputs\screening_master.parquet" --output "bi\outputs\screening_master.xlsx"
```

取り込み後、ダッシュボード側で「最新データを読み直す」を押します。

## ダッシュボード起動

```powershell
cd "C:\Users\mizuk\2026年 investment\Mizuki Fund\bi\pipelines"
python -m streamlit run dashboard_app.py
```

※ 環境によって `streamlit` コマンドが直接通らないため、`python -m streamlit` 推奨。

## 4. 内容のアップデート方法（どこを触るか）

## 画面文言・構成を変えたい

- ホーム文言: `dashboard_app.py`
- 画像レポート画面: `pages/2_値幅レポート_画像.py`
- screening一覧画面: `pages/3_screening_master.py`

## 表示項目・並び・フィルタを変えたい

- `pages/3_screening_master.py` の DataFrame処理を更新
- 例:
  - 表示列の絞り込み
  - 指標フィルタ（例: `OperatingProfit > 0`）
  - ソート軸の切替

## 取得データの中身を変えたい（指標を追加/削除）

- `make_screening_master_v2.py` を更新
- 変更観点:
  - どのAPIを叩くか
  - どの列を最終出力するか
  - 数値変換・欠損処理

## 自動更新条件を変えたい（時刻・曜日）

- Task Scheduler のタスク設定を変更
- 現在想定: 平日（月〜金）19:00

## APIキーの運用を変えたい

- `.env` の `JQUANTS_API_KEY` を更新
- 自動実行は `bi/pipelines/ops/run_make_screening_master_v2.ps1` が `.env` を読む設計

## 5. トラブル時チェックリスト

## ダッシュボードが開かない

- `python -m streamlit --version` でバージョン確認
- 依存不足なら:
  - `python -m pip install -r bi/pipelines/requirements.txt`（リポジトリルートから）または `bi/pipelines` で `pip install -r requirements.txt`

## `screening_master` が空/古い

- `git pull --rebase` の実行ログ確認
- `bi/outputs/screening_master.parquet` の更新時刻確認
- ダッシュボードで「最新データを読み直す」

## 自動更新が動かない

- Task Scheduler の「履歴」「最終実行結果」を確認
- 実行ユーザー権限、作業ディレクトリ、PowerShell実行ポリシーを確認
- `.env` に `JQUANTS_API_KEY` があるか確認
