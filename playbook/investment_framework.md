# 投資フレームワーク全体像

このファイルはMizuki Fundの投資アプローチを一枚岩で定義するドキュメント。
個別ルールは各 `playbook/*.md` を参照し、このファイルは「全体の流れ・仕組み」を規定する。

---

## 1. 分析の3層構造

投資判断は以下の3層を上から順に確認して行う。

```
[ 層1 ] 全体市場（マクロ）
         ↓
[ 層2 ] セクター・競合
         ↓
[ 層3 ] 個別銘柄
```

上位の層が逆風の時は、下位層がどれだけ良くてもポジションサイズを落とす。
全体市場が強い局面で、資金が集まっているセクターの割安銘柄を狙うのが基本姿勢。

---

## 2. 各層の情報収集方法

### 層1：全体市場（マクロ）

**目的**: 日経平均・ダウの動きに関わるあらゆる情報を把握する。

**情報源**:
- 外部 deep research（Perplexity / Gemini Deep Research 等）→ Notion に保存
- Claude がNotionから読み取り、`market/daily/` にMarkdown蓄積

**更新頻度**: 週1〜必要に応じて（日次は不要、週次で大局観を更新）

**保存形式**: `market/daily/YYYY-MM-DD_macro.md`（`→ CLAUDE.md` のデータ鮮度ルール準拠）

---

### 層2：セクター・競合

**目的**: 注目銘柄が属するセクターの資金動向・競合状況を把握する。

**競合の定義方法**（毎回 deep research で特定）:
- 有報の「事業の概要」に記載される競合・同業他社
- 同一OEM向けに同種部品を供給するメーカー
- ETLのSector33Code（J-Quants）が同一の銘柄群

**情報源**:
- 外部 deep research → Notion 保存 → Claude 統合
- EDINET有報（セグメント情報・競合記述）
- J-Quants ETL（同セクター銘柄の株価・指標比較）
- 立花証券API（ニュース取得：未実装）

**更新頻度**: 注目銘柄エントリー時・決算後・相場環境変化時

**保存形式**: `research/sectors/{sector_name}_{YYYY-MM}.md`

---

### 層3：個別銘柄

**目的**: エントリー可否判断に必要な情報をフルセットで揃える。

**スクリーニング方法（3つのルート）**:
1. PMがチャートを見て持ち込む
2. ETL（J-Quants）でスクリーニング → `bi/outputs/screening_master.parquet`
3. Claude に依頼して候補を抽出する

**Deep Dive レポートの構成（統合レポート）**:

| データ | ソース | ツール |
|--------|--------|--------|
| 財務データ（XBRL）| EDINET API | `bi/pipelines/deep_dive.py` |
| 有報テキスト（リスク・MD&A・セグメント）| EDINET API PDF | 同上 |
| 定量指標（PER/PBR/信用残/出来高）| J-Quants ETL | `screening_master.parquet` |
| 独自調査・マクロ文脈 | 外部 deep research | Notion → Claude |
| ニュース | 立花証券API（未実装）| — |
| 投資哲学との照合 | `playbook/` | Claude |

**保存形式**: `research/stocks/{code}_{YYYY-MM-DD}_deepdive.md`

## 3. Perplexity Deep Research 連携フロー

### 基本思想

情報ソースを2つに分けて管理する：

| 情報の種類 | 保存場所 | Claudeの取得方法 |
|-----------|---------|----------------|
| Perplexity Deep Researchの出力 | **MDファイル直接** | `Read` ツールで即読み込み |
| PMが独自に調べた内容（MD化していないもの）| **Notion** | MCP経由で取得 |

```
PM「{銘柄コード}見てる」 or 「{テーマ}を調べたい」
         ↓
Claude: 既存情報を確認（research/ のMD + Notion + ETL + EDINET）
         ↓
Claude: 不足情報を特定し、Step1〜4のプロンプトを順番に出力
         ↓
PM: 各プロンプトをPerplexityに投げ
    → 出力をMDファイルとして research/ に保存
    → PM独自の調査・メモは Notion に保存
         ↓
Claude: MD + Notion + ETL + EDINET を統合してDeep Diveレポート生成
         ↓
（不明点・矛盾があれば）Claude: 追加プロンプトを提案
```

### ファイル保存場所

```
research/stocks/
  {code}_{date}_perplexity_macro.md    ← Step1（全体市場）
  {code}_{date}_perplexity_sector.md   ← Step2（セクター・競合）
  {code}_{date}_perplexity_stock.md    ← Step3（個別銘柄）
  {code}_{date}_deepdive.md            ← Claude統合レポート（最終成果物）

research/sectors/
  {sector}_{YYYY-MM}_perplexity.md     ← セクター単体で調べた場合

market/daily/
  {date}_macro.md                      ← 全体市場単体で調べた場合
```

---

### いつプロンプトを出すか（Claude がトリガーを判断）

| トリガー | 必須Step | スキップ可能Step |
|---------|---------|----------------|
| 新規注目銘柄（初回）| Step1〜4 すべて | なし |
| 決算発表後（既保有）| Step3（個別）, Step2（競合）| Step1は直近7日以内にあればスキップ |
| 相場急変（日経±3%超）| Step1（全体市場）| Step2〜4 |
| 月次レビュー | Step1, 保有銘柄のStep3 | Step2〜4 |
| 継続保有の根拠確認 | Step3（個別）| Step1〜2 |

### Step 1：全体市場プロンプト

**目的**: 日経平均・ダウに影響する現在のマクロ環境を把握する。
**更新条件**: 直近7日以内のNotionレポートがあればスキップ。

```
【Perplexity Deep Research プロンプト】

# 日本株・全体市場の現状分析

以下の観点で、現在（{YYYY年MM月}時点）の日本株市場・グローバルマクロ環境を
詳しく調査・分析してください。日本のスイングトレーダー視点でまとめてください。

## 調査項目

1. **日経平均・TOPIXのトレンド**
   - 直近1ヶ月の動き、主要な転換点とその背景
   - 現在のトレンド方向（上昇・横ばい・下降）

2. **米国市場（ダウ・S&P500・NASDAQ）**
   - 現在の状況と日本株への影響経路
   - FRBの金融政策動向と市場の反応

3. **リスクオン・リスクオフの状況**
   - 現在どちらの局面か、その根拠
   - 円相場（ドル円）の動向と影響

4. **日本国内の注目材料**
   - 日銀の金融政策動向
   - 足元で資金が集まっているセクター（理由も含めて）
   - 足元で資金が逃げているセクター（理由も含めて）

5. **今後1〜2ヶ月の注目イベント**
   - 決算シーズン、政策イベント、地政学リスク等

## 出力フォーマット

- 各項目に見出しをつけてMarkdown形式で出力
- 数値・データには必ず情報源（URL or 媒体名）を付記
- 最後に「スイングトレーダーへの示唆」を3〜5行で要約
- 対象期間（from〜to）を冒頭に明記
```

**保存先**: `market/daily/{date}_macro.md` に貼り付けて保存

---

### Step 2：セクター・競合プロンプト

**目的**: 注目銘柄のセクター動向と主要競合の状況を把握する。
**前提**: Step1が完了していること。

```
【Perplexity Deep Research プロンプト】

# {セクター名}セクター・競合分析

## 前提情報（調査済み）
- 対象銘柄: {銘柄コード} {銘柄名}
- 主要製品・サービス: {有報から取得した事業概要1〜2行}
- 主要顧客: {例：日産・ホンダ向け売上71.7%}
- 地域構成: {例：北米53%・日本24%・アジア23%}

## 調査項目

1. **セクター全体の動向**
   - {セクター名}セクターの直近1〜3ヶ月の株価・業績トレンド
   - セクターに資金が集まっているか、逃げているか（理由）
   - 業界を動かしている主要テーマ・カタリスト

2. **競合他社の特定と状況**
   - 日本国内の主要競合（上場企業）を列挙し、直近の業績・株価状況を説明
   - {銘柄名}と競合他社のポジショニング比較（強み・弱み）
   - 競合他社に対して割安か割高か（PER・PBR比較）

3. **業界構造リスク**
   - 主要顧客（{主要顧客名}）の業績・生産計画の動向
   - 原材料・部品コストの動向
   - 規制・政策変化の影響（関税・環境規制等）

4. **セクター内での{銘柄名}の位置づけ**
   - シェア・競争優位性
   - セクターの資金動向から見て今が買い場か否か

## 出力フォーマット

- Markdown形式、各項目に見出し
- 数値・データには情報源（URL or 媒体名）を付記
- 競合比較は表形式で（銘柄名・時価総額・PER・PBR・最新業績）
- 最後に「このセクターで今注目すべき銘柄Top3」を理由付きで
- 対象期間（from〜to）を冒頭に明記
```

**保存先**: `research/stocks/{code}_{date}_perplexity_sector.md` に貼り付けて保存

---

### Step 3：個別銘柄プロンプト

**目的**: EDINET・ETLで取れない定性情報・最新ニュース・市場評価を取得する。
**前提**: Step1・2が完了し、EDINET deep_dive（`bi/pipelines/deep_dive.py`）を実行済みであること。

```
【Perplexity Deep Research プロンプト】

# {銘柄コード} {銘柄名} 個別銘柄 Deep Research

## 前提情報（調査済み・確認不要）
以下は有価証券報告書・J-Quantsから取得済みのため、数値の再確認は不要。
調査済み情報と矛盾する情報があれば指摘してください。

- 直近本決算: {決算期} 売上高{X}億円 / 営業利益{X}億円 / 純利益{X}億円
- 自己資本比率: {X}% / 有利子負債依存度: {X}%
- 主要リスク（有報記載）: {リスク1}, {リスク2}, {リスク3}
- 現在株価: {X}円 / 時価総額: {X}億円

## 調査項目

1. **最新ニュース・開示情報**（直近3ヶ月）
   - 決算発表・業績修正・経営戦略の発表
   - 主要顧客・サプライチェーンに関するニュース
   - 役員変更・M&A・提携等の企業アクション

2. **アナリスト・市場評価**
   - アナリストカバレッジとレーティング（あれば）
   - 機関投資家の動向（大量保有報告等）
   - SNS・個人投資家コミュニティでの評判・注目度

3. **中期経営計画の進捗確認**
   - 公表されている中計の目標値と現状の乖離
   - 経営陣が直近の決算説明会で強調したポイント

4. **株価のカタリスト候補**
   - 今後3〜6ヶ月で起こりうるポジティブ・ネガティブイベント
   - 類似状況の過去事例（業績回復局面の株価推移等）

5. **需給・テクニカルの補足情報**
   - 信用買残の最近のトレンド（増加・減少）
   - 機関投資家の空売り残の動向

## 出力フォーマット

- Markdown形式、各項目に見出し
- 情報源（URL or 媒体名）を各データに付記
- 「調査済み情報との矛盾点」があれば冒頭に明記
- 最後に「投資判断への示唆」を5行以内で
- 対象期間（from〜to）を冒頭に明記
```

**保存先**: `research/stocks/{code}_{date}_perplexity_stock.md` に貼り付けて保存

---

### Step 4：追加プロンプト（Claude が自動判断）

Notionのレポートを読み込んだ後、以下の状況で追加プロンプトを生成する：

| 状況 | 追加調査の内容 |
|------|--------------|
| 数値がEDINET/ETLと乖離（±10%超）| 「正しい数値を再確認して情報源を明記」 |
| 競合比較が不十分 | 「{具体的な競合名}と{銘柄名}の詳細比較」 |
| カタリストが不明瞭 | 「{具体的な不明点}についてシナリオ別に深掘り」 |
| 地政学・マクロの影響が未分析 | 「{具体的なリスク}が{銘柄名}の業績に与える定量的影響」 |

---

### Deep Research 品質チェック（Claude が実施）

Notionからレポートを読み込んだ際に自動チェック：

| チェック項目 | 基準 |
|------------|------|
| 情報源の明記 | URLまたは媒体名がない数値は「要確認」フラグ |
| EDINET数値との整合 | ±10%超の乖離があれば即指摘 |
| 鮮度 | 対象期間が7日超 → 「古い情報あり」と明示 |
| 矛盾の有無 | Step間で矛盾する情報は統合レポートで明記 |

## 4. トレード記録フォーマット

`portfolio/trade_log.md` に以下の形式で記録する。
エントリー・継続保有・イグジットのすべてで記録を残す。

```markdown
## {銘柄コード} {銘柄名} - {記録種別：エントリー / 継続保有 / イグジット}
**日付**: YYYY-MM-DD
**株価**: XXX円
**ポジション**: XX株（総資産の約X%）

### 根拠
- **全体市場**: （その時の市場環境）
- **セクター**: （業界・競合の状況）
- **ファンダメンタルズ**: （割安根拠・財務状況）
- **テクニカル**: （移動平均・ボリンジャーバンドの状況）
- **需給**: （信用買残/発行株比率、解消日数、トレンド方向）
- **カタリスト**: （期待している材料）

### リスク認識
- （エントリー時点で認識しているリスク）

### 損切りライン
- XXX円（理由：）

### 結果（イグジット時のみ）
- **イグジット株価**: XXX円
- **損益**: +XX% / ▲XX%
- **イグジット理由**: 
- **学び**: 
```

## 5. 改善ループ

### 月次レビュー（月末）

1. その月のトレード記録を振り返る
2. エントリー根拠と実際の値動きを照合
3. `playbook/mistakes_lessons.md` を更新
4. 翌月の注目セクター・銘柄候補をリストアップ

### 蓄積されるもの

| 蓄積物 | 場所 | 使い道 |
|--------|------|--------|
| トレード根拠と結果 | `portfolio/trade_log.md` | 月次振り返り・失敗パターン特定 |
| 市場レポート | `market/daily/` | 大局観の時系列把握 |
| 銘柄Deep Dive | `research/stocks/` | 再エントリー時の参照 |
| セクターレポート | `research/sectors/` | 業界サイクルの把握 |
| 失敗と学び | `playbook/mistakes_lessons.md` | ルール改善のインプット |
| Notion deep research | 投資2026 Mizuki Fund DB | 外部情報の一次ソース |
